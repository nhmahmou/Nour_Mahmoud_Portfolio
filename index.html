A* Pathfinding Visualizer

Draw walls, place a start and goal, then watch A* find the shortest path in real time. Built from the same algorithm as the Java robot agent in CSC 411.

A* SearchJavaScriptInteractive
Click / Drag Draw walls Run A* Find path Clear path Keep walls Random maze Auto-generate Speed slider controls animation
Each cell gets a score: f = g + h

g = exact steps taken from the start to reach this cell. h = estimated steps still needed to reach the goal (Manhattan distance - no diagonals). A* always expands the cell with the lowest f = g + h next. That means it balances "how far did I travel?" against "how far do I still have to go?" - favouring cells that are both reachable cheaply and pointed toward the goal.

Frontier vs explored (yellow vs purple)

The yellow cells are the frontier - cells discovered but not yet fully processed. Each iteration, A* picks the lowest-f cell from the frontier, moves it to explored (purple), and adds its neighbours to the frontier. Notice how the frontier is pushed toward the goal, skipping large parts of the grid that would only make the path longer.

Why the heuristic must not overestimate

If h always underestimates (or matches) the true remaining cost, A* is guaranteed to find the shortest path. If h ever overestimates, A* might dismiss a cheap route as "too far" and return a longer one. Manhattan distance on a grid never overestimates - so the path shown is always optimal.

Connect Four: Minimax AI

Play against an AI that looks 5 moves ahead using minimax with alpha-beta pruning. It assumes you play perfectly and picks the move that wins even in the worst case. Drop a piece by clicking a column.

MinimaxAlpha-BetaAdversarial Search
What is Minimax?

The AI builds a tree of every possible move and countermove up to 5 turns deep. On its turns it picks the move that maximizes its score; on your turns it assumes you pick the move that minimizes it. The result: an agent that plays safe, aggressive, or defensive depending purely on board state.

Alpha-beta pruning

Without pruning, depth-5 search would evaluate thousands of useless positions. Alpha-beta cuts off branches the moment it knows they can't possibly improve the result - reducing the search tree by up to 50% with zero impact on the final answer.
